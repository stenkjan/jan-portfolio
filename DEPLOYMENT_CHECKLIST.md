### 4. Google Analytics 4 Setup

1. Go to https://analytics.google.com
2. Click "Admin" ‚Üí "Create Property"
3. Name: "Jan Stenk Portfolio"
4. Timezone: Europe/Berlin
5. Currency: EUR
6. Create Data Stream ‚Üí Web
7. URL: https://jan-stenk-portfolio.vercel.app
8. Copy the Measurement ID (G-XXXXXXXXXX)

#### Update Code
In `/workspace/src/app/layout.tsx`, replace:
```typescript
gtag('config', 'G-XXXXXXXXXX');
```
With your actual measurement ID.

Or add as environment variable:
```bash
NEXT_PUBLIC_GA4_MEASUREMENT_ID=G-XXXXXXXXXX
```

And update the code to use:
```typescript
gtag('config', process.env.NEXT_PUBLIC_GA4_MEASUREMENT_ID);
```

## üêõ Troubleshooting

### Images Not Loading
**Problem**: Project images return 404
**Solution**: 
1. Verify images uploaded to Vercel Blob
2. Check blob URL in `blobStorage.ts`
3. Verify image names match exactly (case-sensitive)

### Chatbot Not Responding
**Problem**: Chatbot sends messages but no response
**Solution**:
1. Check API route `/api/chat` is deployed
2. Check browser console for errors
3. Verify no CORS issues

### Language Switcher Not Persisting
**Problem**: Language resets on page reload
**Solution**:
1. Check localStorage is enabled
2. Verify `LanguageProvider` wraps entire app
3. Check browser privacy settings

### Google Analytics Not Tracking
**Problem**: No data in GA4
**Solution**:
1. Verify measurement ID is correct
2. Check GA4 script loads (Network tab)
3. Disable ad blockers for testing
4. Wait 24-48 hours for data to appear

### Build Failures
**Problem**: Deployment fails during build
**Solution**:
1. Check build logs in Vercel
2. Ensure all dependencies are in package.json
3. Verify no TypeScript errors
4. Test build locally: `npm run build`
